quota 是什么？What is quota?

Quota 指你在一段时间内可以使用的额度或上限，比如“每天可以生成多少次”“每月免费用多少量”。Quota means the amount or limit you can use during a period, such as how many generations per day or how much free usage per month.

  • 免费额度用完后，可能要等待重置或升级付费。After the free quota is used up, you may need to wait for a reset or upgrade.
  • 不同工具的 quota 不一样，所以同一个项目的成本也会不一样。Different tools give different quotas, so the same project can have different costs.

课堂记法：quota 是“这段时间我还能用多少”的额度。Classroom note: quota means “how much I can still use in this period.”

subscription fees 是什么？What are subscription fees?

Subscription fees 指订阅费，也就是按月或按年付费，换取更高额度、更强模型或更多功能。Subscription fees are monthly or yearly payments for higher limits, stronger models, or more features.

  • 它像“会员费”，不一定每次使用都单独扣钱。It is like a membership fee; each use may not be charged separately.
  • 付费前要看清楚：包含哪些模型、多少额度、能不能取消。Before paying, check which models are included, how much quota you get, and whether you can cancel.

课堂记法：subscription fees 是“先买会员，再使用服务”的费用。Classroom note: subscription fees mean “pay for membership, then use the service.”

token 是什么？What is a token?

Token 是模型处理文本的基本计量单位。它不等于一个完整的汉字或英文单词，而是模型把输入和输出拆分后的文字片段。Tokens are the basic units a model uses to process text. They are not the same as one Chinese character or one English word; they are text pieces created when the model splits input and output.

  • 输入和输出都会计算 token。Both input and output count as tokens.
  • 问题越长、回答越长、代码越多，token 消耗越高。Longer questions, longer answers, and more code use more tokens.
  • 很多 按每 100 万 token 计费，所以 token 会影响成本。Many APIs charge per 1 million tokens, so tokens affect cost.

课堂记法：token 是衡量 AI 读写了多少文字的单位。Classroom note: a token is a unit for measuring how much text the AI reads and writes.

API 是什么？What is an API?

API 是“应用程序接口”。你可以把它理解成网页或软件调用 AI 服务的正式入口：网页把请求发给 API，API 再把 AI 的结果送回来。API means Application Programming Interface. It is the official doorway a webpage or app uses to call an AI service: the webpage sends a request to the API, and the API returns the AI result.

  • 普通用户通常在聊天框里使用 AI。Regular users often use AI in a chat box.
  • 开发者会用 API 把 AI 接进自己的网站、App 或工具里。Developers use APIs to connect AI to their own websites, apps, or tools.

课堂记法：API 是“我的网页去找 AI 帮忙的通道”。Classroom note: an API is the channel my webpage uses to ask AI for help.

iterate 是什么？What does iterate mean?

Iterate 指一轮一轮改进作品。它不是全部推倒重做，而是看结果、找问题、提出更具体的修改要求，再检查新结果。Iterate means improving a work round by round. It does not mean starting over; it means checking the result, finding a problem, asking for a specific change, and checking again.

  • 第 1 轮：先让作品能跑起来。Round 1: make the work run.
  • 第 2 轮：让它更像真实产品。Round 2: make it feel more like a real product.
  • 第 3 轮：修细节、查安全、准备展示。Round 3: polish details, check safety, and prepare to present.

课堂记法：iterate = 发现一个问题，改一轮，留下前后对比。Classroom note: iterate = find one problem, improve one round, and keep before/after evidence.

prompt 是什么？What is a prompt?

Prompt 就是你给 AI 的任务说明。它不只是“问一句话”，而是告诉 AI：目标是什么、用户是谁、输入输出是什么、有什么限制。A prompt is the instruction you give to AI. It tells the AI the goal, user, input, output, and limits.

课堂记法：prompt 是“给 AI 的任务单”。Classroom note: a prompt is a task brief for AI.

prototype 是什么？What is a prototype?

Prototype 是产品原型，也就是还没完全做完、但已经能让别人看懂和试用的早期版本。它用来验证想法，不是最终成品。A prototype is an early version of a product that people can understand and try. It tests an idea before the final product is built.

课堂记法：prototype = 先做一个能试的版本。Classroom note: prototype = a version people can try first.

demo 是什么？What is a demo?

Demo 是演示版，重点是让别人快速看到“这个想法大概怎么工作”。它可以很粗糙，但必须能展示核心功能。A demo is a demonstration version. It quickly shows how an idea works, even if it is not polished yet.

课堂记法：demo = 给别人看的功能演示。Classroom note: demo = a feature shown to others.

Git 是什么？What is Git?

Git 是程序员常用的版本管理工具。它会记录项目每一次重要修改，方便回看、比较、恢复，也方便多人一起做同一个项目。Git is a version-control tool. It records important changes so people can review, compare, restore, and collaborate.

课堂记法：Git 像项目的“存档和时间线”。Classroom note: Git is like a project's save history and timeline.

CSV 是什么？What is CSV?

CSV 是一种很简单的数据表文件。它像 Excel 表格，但本质是纯文字，常用来存一行一行的数据。CSV is a simple data-table file. It is like a spreadsheet saved as plain text.

课堂记法：CSV = 用文字存表格。Classroom note: CSV = a table stored as text.

JSON 是什么？What is JSON?

JSON 是网页和程序很常用的数据格式。它用“名字: 内容”的结构保存信息，机器读起来很方便。JSON is a common data format for webpages and programs. It stores information in name-value structures that machines can read easily.

课堂记法：JSON = 让网页容易读取的数据包。Classroom note: JSON = a data package that webpages can read.

TSV 是什么？What is TSV?

TSV 和 CSV 很像，也是一种表格数据文件。区别是：TSV 用 Tab 分隔每一列，CSV 常用逗号分隔每一列。TSV is similar to CSV. It stores table data, but uses tabs between columns, while CSV often uses commas.

课堂记法：TSV = 另一种用文字存表格的格式。Classroom note: TSV = another text format for tables.

.command 文件是什么？What is a .command file?

.command 是 macOS 上可以双击运行的命令文件。它常用来把一串重复操作打包成一个按钮。A .command file is a macOS command file you can double-click to run. It turns repeated steps into one clickable action.

课堂记法：.command = 电脑里的“一键执行按钮”。Classroom note: .command = a one-click action button on the computer.

debug / 调试 是什么？What does debug mean?

Debug 指找到程序哪里出错，并一步步修好。它不是随便改，而是先看现象，再找原因，再验证修复有没有成功。Debug means finding what is wrong in a program and fixing it step by step: observe, find the cause, and verify the fix.

课堂记法：debug = 找 bug 的原因并修好。Classroom note: debug = find why a bug happens and fix it.

bug 是什么？What is a bug?

Bug 是程序里的错误或异常现象，比如按钮点了没反应、页面显示错、数据没有保存。A bug is an error or unexpected behavior in a program, such as a broken button, wrong display, or unsaved data.

课堂记法：bug = 作品没有按预期工作。Classroom note: bug = the work does not behave as expected.

HTML / CSS / JS 是什么？What are HTML / CSS / JS?

这是网页的三件套：HTML 管内容结构，CSS 管视觉样式，JS 管点击后的行为。They are the three core parts of a webpage: HTML controls structure, CSS controls style, and JS controls behavior.

课堂记法：HTML 是骨架，CSS 是衣服，JS 是动作。Classroom note: HTML is the skeleton, CSS is the outfit, and JS is the action.

HTML 是什么？What is HTML?

HTML 负责网页上“有什么”：标题、段落、按钮、输入框、图片区域和结果区都先由 HTML 放出来。HTML defines what exists on a webpage: titles, paragraphs, buttons, inputs, images, and result areas.

课堂记法：HTML = 网页骨架。Classroom note: HTML = webpage skeleton.

CSS 是什么？What is CSS?

CSS 负责网页“长什么样”：颜色、字体、间距、圆角、阴影、动画和不同屏幕尺寸的排版。CSS controls how a webpage looks: color, type, spacing, corners, shadows, animation, and responsive layout.

课堂记法：CSS = 网页的视觉设计规则。Classroom note: CSS = visual design rules for a webpage.

JavaScript / JS 是什么？What is JavaScript / JS?

JavaScript 简称 JS，负责网页“会做什么”：点击按钮、检查输入、调用 AI、显示结果、复制和保存。JavaScript, or JS, controls what a webpage can do: clicks, input checks, AI calls, results, copy, and save.

课堂记法：JS = 网页的动作和逻辑。Classroom note: JS = webpage actions and logic.

Tailwind 是什么？What is Tailwind?

Tailwind 是一个 CSS 工具库，可以用很多短类名快速写样式。它很快，但如果只堆类名，页面也可能变得难维护。Tailwind is a CSS utility library. It uses short class names to style quickly, but too many classes can become hard to maintain.

课堂记法：Tailwind = 快速写样式的工具箱。Classroom note: Tailwind = a quick styling toolbox.

responsive / 响应式 是什么？What does responsive mean?

响应式设计指网页会根据屏幕大小自动调整排版。电脑、平板、手机打开时，都应该能看清、能操作。Responsive design means a webpage adjusts to different screen sizes, so it works on laptops, tablets, and phones.

课堂记法：响应式 = 换屏幕也不乱。Classroom note: responsive = the layout stays usable on different screens.

component / 组件 是什么？What is a component?

组件是可以重复使用的一小块界面或功能，比如按钮、卡片、输入框。做成组件后，页面会更统一，也更容易修改。A component is a reusable piece of interface or function, such as a button, card, or input. Components keep pages consistent and easier to change.

课堂记法：组件 = 可以重复使用的页面零件。Classroom note: component = reusable page part.

design system / 设计系统 是什么？What is a design system?

设计系统是一套统一规则，规定颜色、字体、按钮、卡片、间距和交互状态。它让多个页面看起来像同一个产品。A design system is a shared rule set for colors, type, buttons, cards, spacing, and states. It makes many pages feel like one product.

课堂记法：设计系统 = 让页面统一的规则书。Classroom note: design system = the rulebook that keeps pages consistent.

src/ 和 scripts/ 是什么？What are src/ and scripts/ folders?

它们通常是项目里的代码文件夹。src/ 多放网站核心代码，scripts/ 多放自动处理、同步、发布这类辅助脚本。They are code folders in a project. src/ often stores the main site code, while scripts/ stores helper scripts for automation, syncing, or publishing.

课堂记法：src/ 是主工程区，scripts/ 是工具区。Classroom note: src/ is the main work area; scripts/ is the tool area.

pipeline / 管线 是什么？What is a pipeline?

Pipeline 指一串固定步骤连起来自动执行，比如同步、测试、构建、发布。它的作用是减少重复劳动和漏步骤。A pipeline is a connected set of steps that run in order, such as syncing, testing, building, and publishing.

课堂记法：pipeline = 把重复流程排成一条自动路线。Classroom note: pipeline = a repeatable route for work steps.

build / 构建 是什么？What does build mean?

Build 指把代码整理、打包成浏览器或服务器可以正式使用的版本。它像把草稿整理成可交付文件。Build means preparing and packaging code into a version that a browser or server can use.

课堂记法：build = 把代码打包成可发布版本。Classroom note: build = package code for release.

deploy / 部署 是什么？What does deploy mean?

Deploy 指把做好的网站或应用放到线上，让别人能通过链接访问。Deploy means putting a finished website or app online so others can open it with a link.

课堂记法：deploy = 把作品发布到网上。Classroom note: deploy = publish the work online.

deepfake / 深伪 是什么？What is a deepfake?

Deepfake 指用 AI 伪造看起来很像真实人物的图片、声音或视频。它可能用于娱乐，也可能造成误导和伤害。A deepfake is AI-made media that imitates a real person's face, voice, or video. It can entertain, but it can also mislead or harm.

课堂记法：deepfake = 像真的 AI 假内容。Classroom note: deepfake = AI fake content that looks real.

watermark / 水印 是什么？What is a watermark?

Watermark 是放在图片或视频上的标记，用来说明来源、版权或“这是 AI 生成内容”。A watermark is a mark on an image or video that shows source, ownership, or that the content was AI-generated.

课堂记法：watermark = 给内容贴身份标签。Classroom note: watermark = an identity label for content.

privacy / 隐私 是什么？What is privacy?

Privacy 指一个人不想随便公开的个人信息，比如真实姓名、照片、住址、联系方式、账号和聊天内容。Privacy means personal information people should not share casually, such as real names, photos, addresses, contacts, accounts, and chats.

课堂记法：privacy = 不能随便公开的个人信息。Classroom note: privacy = personal information that should not be casually exposed.

变量 2： 质量 Variable 2: quality

好的提示词不是“说得高级”，而是把任务变清楚。 要说清：谁用、为什么用、输入什么、输出什么、不能做什么、做到什么才算好。 A good instruction is not fancy wording. It makes the task clear. should clarify: who uses it, why, input, output, limits, and what counts as good.

一个好 prompt 的 7 块积木

角色、目标、背景、输入、输出格式、限制、安全边界。越像产品说明书，AI 越容易稳定发挥。

给格式，比给形容词更重要

不要只说“高级一点”。要说页面包含哪些区块、按钮怎么叫、生成结果有哪些字段、失败时显示什么。

给例子，能快速校准风格

给一个“输入样例”和“理想输出样例”，模型会更容易明白你想要的是搞笑、友善，还是认真。

不够好的 prompt

帮我做一个高级一点的吐槽网站。
要好看，要有 AI 感。

更好的 prompt

你是产品设计老师。请把“吐槽星球”做成一个给 10-12 岁学生用的漫画生成器。

用户：小学生，想把烦恼变成友善、好笑、可分享的漫画。
输入：一句不超过 80 字的吐槽。
流程：选择语气 -> 点击生成 -> 显示生成中 -> 得到漫画、标题、台词、小建议。
限制：不能攻击真实同学、老师、外貌、家庭；不能收集隐私。
输出：单文件 HTML，适合投影展示，手机也能读。

变量 3：产品细化程度

产品细化，就是把一个“灵感”拆成用户、场景、流程、按钮、状态、结果和失败兜底。 越细化，AI 越像在帮你做产品，而不是随机生成一张网页。 Product detail means breaking an idea into user, context, flow, buttons, states, results, and fallback. The more detailed it is, the more AI helps create a product instead of a random page.

最小产品说明书

谁在什么场景遇到什么问题，输入什么，AI 做什么，输出什么，人怎么检查，失败怎么办。把这 7 句写清楚，原型会立刻像产品。

细化不是加功能

细化是让每个功能更有理由。比如“保存图片”不是因为酷，而是因为学生要把漫画带走、发给家长或放进作品集。

结果区：漫画图、标题、主角台词、友善建议、复制文案、保存图片。
安全区：提醒不要点名攻击真实的人。

网页背后的代码，也不一样

同样是“输入吐槽 -> 生成漫画 -> 展示结果”，背后的代码可以很简单，也可以更像一个完整产品。 我们只看几个关键片段，不背代码。 The same flow: input complaint -> generate comic -> show result. Behind it, the code can be simple or product-like. We will only inspect key snippets, not memorize code.

打开 A 源网页 打开 B 源网页

先看代码的 3 层结构

HTML：页面骨架

决定有哪些屏幕、输入框、按钮、画布和结果区。它像产品的房间布局。

CSS：视觉和排版

决定颜色、间距、动画、响应式和按钮状态。它让网页不像一张默认表格。

JavaScript：行为逻辑

决定点击后发生什么、如何调用 AI、失败怎么办、怎么保存和复制。

再看一眼体量和复杂度

A：学生版 402

约 402 行，1.5 万字符。结构清楚，适合快速做出第一版 demo。

B：老师版 1727

约 1727 行，4.9 万字符。多出来的不是炫技，而是状态、校验、兜底和产品细节。

课堂结论 3 层

好原型通常不是只改一个按钮，而是 HTML、CSS、JS 一起升级。

B 老师版：校验 -> 等待页 -> AI -> 兜底 -> 结果页

async function generate() {
  const text = input.value.trim();
  const allChecked = checks.every((box) => box.checked);
  if (!text) {
    errorMsg.textContent = "先写一句想吐槽的话，再发射。";
    input.focus();
    return;
  }
  if (!allChecked) {
    errorMsg.textContent = "发布前检查还没有全部通过。";
    return;
  }

  showScreen(loadingScreen);
  generateBtn.disabled = true;
  try {
    const result = await requestAiComic(text);
    refreshResult(result);
    showScreen(resultScreen);
  } catch (error) {
    const fallback = createResult(text);
    refreshResult(fallback);
    showScreen(resultScreen);
  } finally {
    generateBtn.disabled = !checks.every((item) => item.checked);
  }
}
差异：A 的逻辑像“按钮触发一个任务”；B 的逻辑像“产品流程”，会检查输入、安全、加载、失败兜底和结果刷新。
